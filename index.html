Proposal Holder Configuration

Use this page to finalize proposal holder specifications before updating manufacturing documentation and requesting final quotations.

ℹ️ Connecting…
Required before saving changes.
Shared-mode setup (Supabase) — read me

This tool saves a single shared configuration. To sync it across everyone who opens the file, add a free Supabase project's credentials in the SUPABASE_CONFIG block near the bottom of this HTML file, then run the SQL below in Supabase → SQL Editor. Until then the page works for you locally (saved in this browser).

1. Create a project at supabase.com. 2. SQL Editor → run:

create table if not exists proposal_holder_configuration (
  id            int primary key default 1,
  material      text,
  logo_included text,
  logo_version  text,
  logo_placement text,
  box_shape     text,
  box_size      text,
  notes         text,
  updated_at    timestamptz default now(),
  updated_by    text,
  constraint single_row check (id = 1)
);
insert into proposal_holder_configuration (id) values (1)
  on conflict (id) do nothing;

alter table proposal_holder_configuration enable row level security;
create policy "shared read"  on proposal_holder_configuration for select using (true);
create policy "shared write" on proposal_holder_configuration for update using (true);

3. Settings → API → copy the Project URL and anon public key into SUPABASE_CONFIG. Save the file. Done — every visitor now reads & writes the same row.

Note on the anon key: it is safe to expose in front-end code; row-level security controls access. For an internal team tool this shared-write policy is intentional.
